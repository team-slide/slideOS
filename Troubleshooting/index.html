Windows Troubleshooting Guide

🚀 Getting Started - Important First Steps

Before using Innioasis Updater for the first time, please follow these essential steps:

  1. Install the necessary drivers - This is crucial for the app to communicate with your Y1 device
  2. Restart your PC - This ensures all drivers are properly loaded and recognized
  3. Follow on-screen instructions carefully - The app will guide you through each step
  4. Ensure your Y1 is connected properly - Use a good quality USB cable

Following these steps will ensure the smoothest System Software (firmware) update or custom firmware install journey.

❄️ App Frozen After Downloading Firmware

If the Innioasis Updater app becomes unresponsive (shows "Not Responding"):

  1. End the task/force quit the app - Use Task Manager (Ctrl+Shift+Esc) to end the process
  2. Disconnect all non-essential USB devices - Keep only your Y1 connected
  3. Restart the app - Launch Innioasis Updater again
  4. Click "Try Again" when the app opens back up
  5. Try a different method if the first attempt doesn't work

🔗 Missing Shortcuts on Windows

If you're missing shortcuts on your desktop or start menu, run this command to restore them:

"%LocalAppData%\Innioasis Updater\pythonw.exe" updater.py --force

Your shortcuts will repopulate after the app updates itself.

🛠️ Windows Toolkit Access

You can fix many issues with Innioasis Updater using the Windows Toolkit:

  1. Press Windows Key + R
  2. Run this command:
explorer.exe "%LocalAppData%\Innioasis Updater\Troubleshooting"

Or download the latest toolkit directly:

Download Windows Toolkit

🔧 Firmware Installation Issues

If firmwares aren't installing with any method or SP Flash Tool/MTKClient in the troubleshooting folder:

  • Make sure the blue Driver Setup button isn't showing in your app
  • If it is, you're missing one of the required drivers for both methods
  • Method 1 and 2 use MTKclient, Method 3 uses SP Flash Tool

💻 Windows App Doesn't Work At All

If the Windows app doesn't work at all, don't worry! You have alternatives:

  1. Download firmwares from firmware.innioasis.app
  2. Extract the zip into the Innioasis Updater folder (right-click start menu shortcut → Open File Location)
  3. Use one of the shortcuts in the Troubleshooting folder

🔄 Fix Stuck Updates

If you have a stuck update:

  1. Reboot your computer
  2. Start Innioasis Updater again from the start menu
  3. It will then update fine, and this shouldn't happen again in the future

If this doesn't work:

  • Right-click on your Innioasis Updater shortcut
  • Look for the "Fix Stuck Updates" folder
  • If you don't have one, download it from troubleshooting.innioasis.app

📱 Y1 Remote Control Features

After installing your first update on the Y1 using Innioasis Updater, you unlock powerful remote control capabilities:

  • Screenshot your Y1 - Capture what's on your device's screen
  • Remote control - Control your Y1 from your computer
  • Test and deploy apps - Install and manage applications remotely

🎵 Rockbox-y1 Integration

There's currently one app available specifically made for the Y1: Rockbox-y1 by the developer. It's available pre-packaged as a ROM from github.com/rockbox-y1/rockbox.

⚙️ MTKclient Installation Method

Windows users can enable MTKclient as an install method (it's usually used for macOS and Linux users and is more fiddly):

  • Windows users only require: MTK SP USB Driver
  • Optional but recommended: Install USBDK to enable further functionality
  • MTKclient provides an alternative installation method if other methods fail

🖥️ ARM64 Windows Users

If you're running Windows on ARM64, you have limited but still useful options:

  • Y1 Remote Control only - After first updating your Y1 or installing custom firmware from firmware.innioasis.app on another computer
  • Alternative option: You may potentially be able to run the full Linux version in WSLg by using USBIPD-win for passthrough (users are on their own with this setup)

⚠️ Important Note

ARM64 Windows users cannot directly install firmware but can use remote control features after the initial setup is done on another computer.

macOS Troubleshooting Guide

🚀 Getting Started - Important First Steps

Before using Innioasis Updater for the first time on macOS:

  1. Install Homebrew - Required for dependencies
  2. Follow on-screen instructions carefully - The app will guide you through each step
  3. Ensure your Y1 is connected properly - Use a good quality USB cable
  4. Grant necessary permissions - macOS may ask for various permissions

Following these steps will ensure the smoothest System Software (firmware) update or custom firmware install journey.

🍎 macOS Specific Troubleshooting

Common issues on macOS and their solutions:

  • App won't open: Right-click the app → Open, or go to System Preferences → Security & Privacy → Allow
  • Permission denied: Grant Terminal/App Store permissions when prompted
  • USB not recognized: Try a different USB port or cable
  • Homebrew issues: Update Homebrew with brew update && brew upgrade

📱 Y1 Remote Control Features

After installing your first update on the Y1 using Innioasis Updater, you unlock powerful remote control capabilities:

  • Screenshot your Y1 - Capture what's on your device's screen
  • Remote control - Control your Y1 from your Mac
  • Test and deploy apps - Install and manage applications remotely

🎵 Rockbox-y1 Integration

There's currently one app available specifically made for the Y1: Rockbox-y1 by the developer. It's available pre-packaged as a ROM from github.com/rockbox-y1/rockbox.

⚙️ MTKclient on macOS

MTKclient is the primary installation method on macOS:

  • No additional drivers needed - macOS handles USB communication natively
  • More reliable - MTKclient works better on macOS than Windows
  • Command-line interface - Provides more control over the installation process

🔒 macOS Security Settings

If you encounter security-related issues:

  1. Go to System Preferences → Security & Privacy
  2. Click "Allow" when prompted about running the app
  3. For command-line tools, you may need to run sudo spctl --master-disable temporarily
  4. Remember to re-enable security with sudo spctl --master-enable after installation
